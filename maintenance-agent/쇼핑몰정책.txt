[상품 등록] 

*상품공통사항 

- 썸네일 이미지 용량은 API 정책상 10MB까지만 등록 가능합니다. 

- 일부 특수문자(\ / : * ? " < >) 및 네이버에서 규정한 금칙어가 포함된 상품명은 사용할 수 없습니다. 

- 재고관리 사용 시, 옵션 관리 코드에 SKU코드를 전송합니다. (세트상품인 경우에는 세트코드로 전송됩니다.) 

- 조합형 옵션 선택 시 옵션가 0원 / 판매 상태 '판매중' / 사용여부 Y인 옵션이 하나라도 있어야 등록됩니다. 

- 독립형 옵션 선택 시 스마트스토어상에 단독형으로 전송되며, 판매수량 및 추가 금액은 반영되지 않습니다. 

- 원산지 국산의 경우 국내 > 지역선택 > 도시선택까지 모두 설정해야 합니다. 

  (판매자센터 내에서는 지역,도시 선택 없이 국산으로만 설정 가능하나 API 미지원으로 솔루션을 통한 등록/수정 시 지역, 도시 모두 선택해야 합니다.)

- 원산지 국내 > 경남 > 마산시 선택 시에 경남 > 창원시 마산합포구로 치환하여 전송됩니다.

- 한글의 경우 글자 당 3byte로 계산됩니다. 

- 상품 상세> 쇼핑몰정보> 추가항목 탭에서 카테고리 별 '상품 속성'을 설정할 수 있으며, 변경하지 않을 경우 기본 항목은 쇼핑몰 정보 유지로 설정됩니다. 

- 친환경 인증을 '[친환경]환경표지'로 등록하고 싶은 경우, 인증정보>인증유형을 '친환경인증-저농약농산물' 로 선택합니다. 

- 상품정보제공고시 추가 입력 사항은 스마트스토어 상품의 [상세] > 쇼핑몰정보 > 추가항목 > 상품정보고시 추가정보에서 제공합니다.(해당 항목은 필수로 입력해 주셔야 하며, 기본 선택 값은 "상품 상세설명 참조"로 제공합니다.)

- 이미지파일의 실제 확장자 파일타입이 파일명과 상이할 경우 실제 파일타입의 확장자로 등록됩니다. 

- 움직이는 이미지(gif)를 등록할 경우 첫 번째 컷의 정지된 이미지로 등록됩니다.

- 어린이 제품 필수 카테고리가 아닌 경우 어린이제품 인증정보 입력 불가 합니다. 



*옵션 추가금액 입력 기준

- 판매가 2,000원 미만인 경우 0~+100%

- 판매가 2,000원~10,000원 미만인 경우 -50%~+100%

- 판매가 10,000원 이상 -50%~+50%까지 입력 가능합니다.



*도서 카테고리 

- 잡지 카테고리를 포함한 일부 카테고리는 API 미지원으로 등록 및 관리가 불가할 수 있습니다. 

- 도서 상품 전송 시 ISBN번호 및 작가/출판사/출간일 정보는 필수 항목입니다. 

- 작가/출판사 입력 경로 : 상품 상세 > 기본정보 > 브랜드/모델명/제조사 (작가=브랜드, 출판사=제조사 입력) 

- 출간일은 상품정보제공고시의 '출간일' 항목을 입력합니다. (상세설명참조로 등록 불가) 

- 도서 카테고리의 경우 상품리뷰 작성 시 지급 포인트를 설정할 수 없습니다. (API 미지원) 



[상품 수정] 

- 재고 0개로 품절된 경우 솔루션상에서 입고 및 재판매 처리가 불가능합니다. SCM에서 입고 처리 후 상품상태동기화 작업을 해주셔야 합니다.

- 스마트스토어 간편옵션은 '마스터상품을 이용한 연동된 쇼핑몰 상품 정보 공통 수정'과 '엑셀 일괄 수정' 작업을 이용한 일괄 수정이 지원되지 않습니다. 상품 '상세 페이지' 또는 '옵션 전용 엑셀 일괄 수정' 메뉴에서 수정해주세요.
 ([색상명 변경]이 된 옵션에 대해서는 '옵션 전용 엑셀 일괄 수정' 작업이 지원되지 않습니다. 기본 제공값으로 작업해주시거나, 상품 [상세]를 통한 개별 작업을 해주셔야 합니다.)

- [독립형]의 옵션값 품절 시 [템플릿] > [상품전송 설정] > [품절옵션 노출 여부] > [노출안함] 으로 설정 해주셔야  옵션값 품절 (사용여부 N 으로 적용, 판매 페이지 미노출) 적용 됩니다. 

- 옵션의 상태를 품절로 변경하여 수정 전송 시 재고를 0으로 전송합니다.

- 스마트스토어 단일상품(그룹상품) 옵션 추가 후 상품  수정 전송 시 [ 템플릿 > 상품전송 설정 > 상세설명 수정여부 [수정안함] ]으로 설정 되어 있어도 그룹상품의 상세설명 에디터-> HTML로 변경됩니다. 

- 판매자 센터 내 오늘출발 가능 옵션으로 설정된 상품을 [옵션없음]으로 부분수정 진행할 경우, 템플릿 내 설정된 [배송속성] 정보로 전송됩니다.

- 카탈로그가 연결된 상품의 경우 브랜드/제조사 수정이 불가합니다. 

- 그룹상품의 경우, 옵션별 이미지 수정은 지원하지 않습니다. (대표이미지 일괄 전송)



[상품 가져오기] 

- SmartEditor ONE으로 작업된 상품의 상세 설명은 HTML소스로 강제변환되며 수정전송 시 HTML 방식으로 변경됩니다. 스마트에디터로 등록한 상세설명으로 사용을 원하시는 경우, 스마트스토어 템플릿 기본 정보의 "상품수정시 상세정보 수정여부"를 "수정 사용 안함"으로 체크해 주셔야 합니다. 

- 제한문자 ( * ? " < > )가 입력된 상품은 상품 가져오기가 지원되지 않습니다. 

- 인증 대상카테고리는 스마트스토어에 인증정보가 입력되어 있어야 상품 가져오기 작업이 가능합니다.(API 제한) 

- 스마트스토어 API에서 정렬을 지원하지 않는 관계로, 추가 구매 옵션은 SCM에서 설정한 순서대로 수집되지 않습니다. 

- 상품정보제공고시가 "식품(농,축,수산물)" 인 상품을 수집할 경우, 솔루션에는 "농산물" 로 매칭되어 저장됩니다. 이후 수정 전송 시에는 쇼핑몰에 선택되어 있는 상품정보제공고시로 유지됩니다. 

- 쇼핑윈도에만 등록되어 있는 상품은 수집되지 않습니다.

- 원산지가 "국산"만 선택되고 하위 정보가 없는 경우 수집 시 "기타> 국산"으로 수집됩니다.

- 상품의 [출고지]항목에 [대표반품교환지], [반품지] 항목에 [대표 상품 출고지] 설정 후 상품 수집 시 템플릿의 [출고지],[반품지] 항목은 공란으로 수집됩니다.

- 그룹상품의 하위 옵션으로 구성된 상품을 일반상품으로 수집하려면 '특정 상품 수집'으로 진행 가능합니다.

- 판매상태 '판매중'인 상품을 수집합니다. (전시상태 '전시중', '전시중지' 상태의 상품도 판매상태가 '판매중'일 경우 모두 수집)

[미지원 항목] 

- API 미지원으로 인해 '6개월 구매 이력' 여부 항목은 수집 불가합니다. 

- API 미지원으로 인해 퀵수집 시에는 결제완료 상태의 주문만 단독으로 수집할 수 없습니다. 

- 판매자 센터에서 운송장을 등록했다가 취소 하여 '집화취소' 처리된 주문 건은  솔루션으로 수집을 지원하지 않습니다. 



[참고 사항] 

- 상품 중 오늘출발이 설정된 주문이 수집될 경우 상품명 좌측에 [오늘출발] 이 표기됩니다.

- 스마트스토어 특성상 공급가 책정이 타 쇼핑몰에 비해 시간이 오래 소요될 수 있으며, 공급가가 책정되기 전에 수집한 주문은 공급가 항목이 0원으로 수집될 수 있습니다. 

- 공급가가 0원인 주문의 공급가 업데이트 방법

SCM에서 해당 주문의 정산 예정금액이 있는지 확인 후 플토에서 주문 삭제 후 재수집. (쇼핑몰에서 배송준비중이 아닌 상태는 삭제 후 재수집이 불가하오니 숙지해 주시기 바랍니다.) 
플토에서 구매결정(정산완료)로 동기화 시, 공급가(정산 예정금액)으로 업데이트 합니다.


[주문 수집/동기화] 

- 스마트스토어 특성상 주문 수집 및 동기화 기간은 11일로 제한하고 있습니다. 

- 쇼핑몰의 주문상태 변경일이 작업일로부터 11일이 경과된 주문은 주문동기화 대상에서 제외됩니다. 

- 윈도 서비스 상품의 주문 수집은 지원되지만 일반 스마트스토어 주문 건과 '쇼핑몰 상품코드' 링크 유형이 다르기 때문에 링크를 통한 상세페이지 보기는 제공되지 않습니다. 

- 주문에 '출고지'가 기재되지 않은 주문건을 제외하고 수집됩니다.

- 주문 수집시 공급가(정산예정금액) 항목에 대해 수집되지 않을 경우 주문 동기화를 통해 업데이트 됩니다.

- 일반 상품으로 등록 > 그룹상품(단일상품)으로 전환 한 경우 주문 수집 시 옵션명이 수집 되지 않아 SKU자동 매칭이 되지않습니다. (상품 삭제 후 '상품가져오기'로 상품 재등록 필요)

- 플토 내 등록된 주문 건 상태가 '취소요청', '교환요청', '교환진행', '반품요청', '반품진행' 일때, 쇼핑몰 상에서 취소철회 등으로 주문 상태가 변경되었을 경우, 주문 수집 작업을 통해 [주문재확인] 상태로 변경됩니다.

- 그룹상품에 [추가문구]를 사용할 시 상품명 앞에 [추가문구]가 기재되어, 주문 수집 시에 SKU상품 자동 매칭이 불가하오니 매칭규칙 생성 후 SKU상품 매칭 바랍니다.

- 결제완료 > 주문재확인 상태로 동기화된 경우 수동으로 주문 상태를 변경하더라도 'SKU상품 자동 매칭'이 불가합니다. SKU수동 매칭 또는 배송처를 지정하여 처리부탁드립니다. 

(SKU상품 자동 매칭은 '신규주문'으로 수집 시 진행됩니다.)
